def hour_utc(timestamp):
    now  = datetime.utcfromtimestamp(timestamp)

def get_timezone(city):
    place,(lat,lon) = do_geocode(city)
    timezone = do_timezone((lat,lon))
    return timezone

def do_geocode(address):
    g = geopy.GoogleV3()
    try:
        return g.geocode(address)
    except GeocoderTimedOut:
        return do_geocode(address)

def do_timezone((lat,lon)):
    g = geopy.GoogleV3()
    try:
        return g.timezone((lat,lon))
    except GeocoderTimedOut:
        return do_timezone((lat,lon))

Обобщая полученные рзультаты исследования, на приведенных выше графиках мы наблюдаем как изменяется активность игроков в полученных часовых поясах. Однако при более внимательном рассмотрении результатов, можно обнаружить, что некоторые страны входят не в "свои" часовые пояса. Если быть точным, то этими странами являются такие страны как США и Канада (бывшие европейские колонии в Северной Америке). Данная ошибка возникает из-за совпадения названий малых населенных пунктов Старого Света и Нового Света. Хочу обратить внимание что в исследовании было проведено удаление таких населенных пунктов, хотя погрешность корректной идентификации страны по населенным пунктам все же остается. Для уменьшения этой погрешности можно в качестве начальной выборки городов использовать те города, в которых количество пользователей наибольшее. Более того, в предедущих иследованиях было выявленно что наибольший трафик идет из крупных городов с большой численностью населения, в таком случае небольшие города имеют слабое воздействие на результаты исследования. (Так изначально были взяты города с количеством пользователей ">25", но при изменении этого условия на ">150" ошибка минимиизируется) Точное значение часовых поясов для стран: https://en.wikipedia.org/wiki/List_of_tz_database_time_zones В конечном итоге мы можем наблюдать наибольшую активность в часовых поясах Северной Америки и Западной Европы.
